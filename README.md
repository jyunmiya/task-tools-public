# WEBサイト品質管理ツール 仕様書

## 1. 概要
`WEBサイト品質管理ツール` は、Google Apps Script と Google スプレッドシートで構成する、Web制作向けの品質管理ツールです。  
案件ごとにチェック対象を作成し、`制作前` `制作` `公開前` `公開後` の4フェーズで進捗を更新します。

このツールは品質管理に必要な最小機能へ絞っており、
- 案件画面では進捗更新
- マスタ管理画面では標準タスクの追加・編集・有効/無効管理

を行います。

## 2. 目的
- 案件ごとの品質確認タスクをフェーズごとに整理して管理する
- 担当者がタスクのステータスと備考を簡単に更新できるようにする
- 標準タスクをマスタで一元管理し、新規案件と既存案件へ反映できるようにする

## 3. 画面構成

### 3-1. 案件画面
- 案件選択
- 新規案件作成
- フェーズ別進捗表示
- フェーズ切替タブ
- カテゴリ別タスク一覧
- タスクごとのステータス更新
- タスクごとの備考更新
- 案件情報表示

### 3-2. マスタ管理画面
- 標準タスク一覧表示
- 標準タスク追加
- 標準タスク編集
- 標準タスクの有効/無効切替
- フィルタ表示
  - すべて
  - 有効のみ
  - 無効のみ

## 4. 運用ルール
- 標準タスクの追加・編集・有効/無効はマスタ管理画面で行う
- 案件画面では、案件に展開済みのタスクに対してステータスと備考を更新する
- 新規案件作成時に、その時点で有効な標準タスクを案件へ展開する
- 新規の有効タスクをマスタへ追加した場合、既存案件へも未登録分を自動追加する
- マスタの有効化時も、既存案件へ未登録分を自動追加する
- マスタ編集内容は既存案件へ自動再反映しない

## 5. データ構造

### 5-1. TaskMaster
標準タスクを管理するシートです。

| 列名 | 内容 |
| --- | --- |
| `task_master_id` | 標準タスクID |
| `phase` | フェーズ |
| `category` | カテゴリ |
| `task_name` | タスク名 |
| `priority` | 重要度 |
| `is_active` | 有効フラグ |
| `sort_order` | 表示順 |
| `updated_at` | 更新日時 |

補足:
- 標準タスクIDは `TM-001` 形式を基本とする
- 新規追加する標準タスクも `TM-xxx` 形式の連番で採番する
- 以前に作成された UUID 形式IDはそのまま保持する

### 5-2. Projects
案件情報を管理するシートです。

| 列名 | 内容 |
| --- | --- |
| `project_id` | 案件ID |
| `project_name` | 案件名 |
| `owner` | 担当者 |
| `current_phase` | 現在フェーズ |
| `status` | 案件ステータス |
| `start_date` | 開始日 |
| `launch_date` | 公開予定日 |
| `created_at` | 作成日時 |

### 5-3. ProjectChecks
案件ごとのタスク進捗を管理するシートです。

| 列名 | 内容 |
| --- | --- |
| `project_id` | 案件ID |
| `task_master_id` | 標準タスクID |
| `status` | ステータス |
| `note` | 備考 |
| `updated_by` | 更新者 |
| `updated_at` | 更新日時 |

補足:
- `project_id + task_master_id` の組み合わせで管理する
- マスタ追加時は、未登録の組み合わせのみ既存案件へ追加する

## 6. ステータスと進捗

### 6-1. ステータス
- `ok`
  - 完了
- `ng`
  - 要確認
- `pending`
  - 未確認
- `na`
  - 対象外

### 6-2. 進捗計算
- `ok` と `na` を完了扱いとする
- フェーズごとの進捗率を計算する
- 案件全体の進捗率も計算する

### 6-3. 通常画面での見せ方
- フェーズカードでは `進捗%` に加えて、`完了 / 未確認 / 要確認` の件数を表示する
- 選択中フェーズ上部でも、`総タスク数 / 完了件数 / 未確認件数 / 要確認件数 / 最終更新` を表示する
- カテゴリ見出しにも `完了 / 未確認 / 要確認` の件数を表示する

### 6-4. タスク行の表示ルール
- 通常時はステータスをバッジ表示する
  - `OK`
  - `未確認`
  - `要確認`
  - `N/A`
- `編集` ボタンを押した行だけ、ステータス選択と備考入力を表示する
- `保存` 後はバッジ表示へ戻し、更新者・更新日時も反映する

## 7. GAS ファイル構成

### `Code.gs`
- `doGet()`
  - Web画面を表示する
- `doPost(e)`
  - API リクエストを振り分ける

### `Checklist.gs`
- `initializeSpreadsheet()`
  - スプレッドシート初期化
  - シート構造作成
  - 標準タスク初期投入
- `getAppState()`
  - 案件一覧と基本状態を返す
- `createProject(payload)`
  - 新規案件を作成し、標準タスクを案件へ展開する
- `saveProjectTaskUpdates(payload)`
  - 案件タスクのステータスと備考を保存する
- `getMasterItems(filter)`
  - 標準タスク一覧を返す
- `saveMasterItem(payload)`
  - 標準タスクを追加または更新する
  - 新規かつ有効なマスタ追加時は、既存案件へ未登録タスクを追加する
- `toggleMasterItem(payload)`
  - 標準タスクの有効/無効を切り替える
  - 有効化した場合は、既存案件へ未登録タスクを追加する
- `getProjectChecklist(projectId)`
  - 案件詳細とフェーズ別タスク一覧を返す
- `upgradeTaskMastersToLatest25()`
  - 既存の `TaskMaster` を最新の25件セットへ置き換える
  - 既存案件の `ProjectChecks` も、最新25件ベースで再生成する

### `index.html`
- 案件画面とマスタ管理画面の UI を提供する
- `google.script.run` を使って GAS 関数を呼び出す

### `appsscript.json`
- Apps Script の設定ファイル

## 10. 現在の標準タスク初期値
新規に `TaskMaster` を初期化する際は、コーポレートサイト向けの品質確認重視タスクを投入します。

### 制作前
- `現状分析`
  - 現行サイトの主要流入経路を把握している
  - 現行サイトの主要ページと導線を把握している
  - 競合3社の導線と訴求軸を比較した
- `要件整理`
  - サイトの目的と主要KPIを整理した
  - ターゲット・訴求内容・優先導線を整理した
- `素材・体制`
  - 必要素材の有無と不足分を確認した
  - 公開までの確認体制と担当者を整理した

### 制作
- `情報設計`
  - サイトマップまたはページ構成を確認した
  - 各ページの役割と導線設計を確認した
- `デザイン・実装`
  - ファーストビューで訴求内容が伝わる
  - 主要導線のCTAが分かりやすく配置されている
  - PCとスマホで崩れず閲覧できる
- `コンテンツ`
  - 見出し構造と本文内容が整理されている
  - 画像・リンク・埋め込み要素が正しく配置されている

### 公開前
- `表示確認`
  - 主要ブラウザで表示崩れがない
  - フォーム・電話・外部リンクが正常に動作する
  - 主要導線が想定どおり遷移する
- `SEO・計測`
  - `title / description / OGP` の基本設定を確認した
  - `GA4 / Search Console` など必要な計測設定を確認した
- `公開設定`
  - `noindex` `ベーシック認証` `テストコード` の解除を確認した
  - `favicon` `404` `SSL` `リダイレクト` など公開設定を確認した

### 公開後
- `初期確認`
  - 公開直後に主要ページの表示と導線を確認した
  - フォーム送信や主要CVが本番環境で動作した
- `引き継ぎ`
  - 更新方法・注意点・運用連絡先を整理した
  - 計測開始と初期監視の確認を完了した

## 11. 標準的な運用フロー
1. `initializeSpreadsheet()` を実行して初期化する
2. マスタ管理画面で標準タスクを確認し、必要に応じて編集する
3. 案件画面で新規案件を作成する
4. 担当者が各タスクのステータスと備考を更新する
5. 新規マスタを追加した場合は、必要に応じて既存案件への反映を確認する

## 12. 補足
- 本ツールは品質管理に特化した軽量版である
- `Dashboard` シート、ダッシュボード更新、レポート出力、メール送信は現行仕様に含めない
- `initializeSpreadsheet()` では既存 `TaskMaster` は上書きしない
- 既存環境を最新の標準タスク25件へ揃える場合は `upgradeTaskMastersToLatest25()` を使う
