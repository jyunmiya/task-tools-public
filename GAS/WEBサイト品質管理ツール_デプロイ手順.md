# WEBサイト品質管理ツール デプロイ手順

## 1. この手順書でできること
この手順書は、Google Apps Script で `WEBサイト品質管理ツール` を公開するための手順をまとめたものです。  
初めてセットアップする場合と、すでに使っている環境を最新の標準タスク25件へ更新する場合を分けて説明しています。

このツールは、次の4ファイルで動作します。
- `Code.gs`
- `Checklist.gs`
- `index.html`
- `appsscript.json`

作成されるシートは次の3つです。
- `TaskMaster`
- `Projects`
- `ProjectChecks`

## 2. 初回セットアップ手順

### 2-1. Apps Script プロジェクトを作成する
1. ブラウザで Google Apps Script を開く
2. `新しいプロジェクト` を作成する
3. プロジェクト名を `WEBサイト品質管理ツール` など分かりやすい名前に変更する

### 2-2. ファイルを登録する
Apps Script に次の4ファイルを登録します。

1. `Code.gs` を作成または上書きして内容を貼り付ける
2. `Checklist.gs` を作成して内容を貼り付ける
3. `index.html` を作成して内容を貼り付ける
4. `appsscript.json` を表示して内容を貼り付ける

確認ポイント:
- ファイル名が正しいこと
- 保存が完了していること
- 構文エラーが出ていないこと

### 2-3. 初期化を実行する
1. 関数一覧から `initializeSpreadsheet` を選ぶ
2. `実行` をクリックする
3. 初回だけ Google の認証画面が出たら許可する

`initializeSpreadsheet()` を実行すると、次が自動で作成されます。
- スプレッドシート
- `TaskMaster` `Projects` `ProjectChecks` の3シート
- 標準タスクの初期データ

補足:
- 標準タスクの初期投入は、`TaskMaster` が空のときだけ行われます
- すでに `TaskMaster` にデータがある場合、既存データは上書きされません

### 2-4. 初期化後に確認する
初期化が終わったら、次を確認してください。

1. スプレッドシートが作成されている
2. シートが3つある
   - `TaskMaster`
   - `Projects`
   - `ProjectChecks`
3. `TaskMaster` に標準タスクが入っている
4. 標準タスクIDが `TM-001` 形式で入っている

## 3. Webアプリとして公開する

### 3-1. デプロイする
1. Apps Script 右上の `デプロイ` をクリックする
2. `新しいデプロイ` を選ぶ
3. 種類で `ウェブアプリ` を選ぶ
4. 必要な公開設定を行う
5. `デプロイ` をクリックする

### 3-2. 公開URLを確認する
デプロイが完了すると URL が表示されます。  
その URL を開くと、品質管理ツールの画面が表示されます。

更新時の注意:
- コードを修正しただけでは公開画面は更新されません
- 修正後は `デプロイを管理` から再デプロイしてください

## 4. 初回動作確認

### 4-1. 画面が開くか確認する
- 案件画面が表示される
- `案件画面` と `マスタ管理` を切り替えられる
- `スプレッドシートを開く` リンクが表示される

### 4-2. 案件を作成できるか確認する
1. 案件名を入力する
2. 担当者や日付を入力する
3. `新規案件作成` を実行する

確認ポイント:
- `Projects` に案件が追加される
- 作成した案件が画面の一覧に出る
- `ProjectChecks` に標準タスク分の行が追加される

### 4-3. タスクを更新できるか確認する
1. 任意の案件を開く
2. 編集したい行の `編集` を押す
3. ステータスや備考を変更する
4. `保存` を実行する

確認ポイント:
- 通常時はステータスがバッジ表示される
- 編集中の行だけプルダウンと備考入力欄が出る
- 保存後はバッジ表示へ戻る
- `ProjectChecks` の `status` `note` `updated_by` `updated_at` が更新される
- カテゴリ見出しの `完了 / 未確認 / 要確認` 件数も更新される

### 4-4. マスタ管理を確認する
1. `マスタ管理` 画面へ切り替える
2. 標準タスク一覧が表示されることを確認する
3. 必要なら新規タスクを追加する

確認ポイント:
- 新規マスタIDが `TM-xxx` 形式で採番される
- 有効な新規タスクは既存案件にも追加される

## 5. 既存環境を最新の標準タスク25件へ更新する場合
この手順は、すでに運用している環境を最新の25件セットへ揃えたい場合だけ実行します。  
初回セットアップだけなら不要です。

### 5-1. 実行手順
1. Apps Script で `Checklist.gs` を最新版にする
2. 関数一覧から `upgradeTaskMastersToLatest25` を選ぶ
3. `実行` をクリックする
4. 実行後に Web アプリを再読み込みする

### 5-2. この更新で起こること
- `TaskMaster` が最新25件セットに置き換わる
- 既存案件の `ProjectChecks` も最新25件ベースで再生成される

### 5-3. 注意点
- 旧タスクに対する進捗は引き継がれません
- 既存案件は、最新25件で再スタートする前提です
- 実行前に必要であればシートのバックアップを取ってください

## 6. よくあるつまずき

### 6-1. `initializeSpreadsheet` が見つからない
- `Checklist.gs` が登録されているか確認する
- 保存できているか確認する
- 関数一覧を開き直す

### 6-2. 案件を作成しても画面に出ない
- Web アプリを再デプロイしているか確認する
- 公開 URL を再読み込みする
- シークレットウィンドウでも確認する

### 6-3. マスタを追加しても案件画面に出ない
- マスタが `有効` になっているか確認する
- 追加後に画面を再読み込みする
- `ProjectChecks` に対象の `task_master_id` が追加されているか確認する

### 6-4. 最新25件にしたいのに反映されない
- `initializeSpreadsheet()` では既存 `TaskMaster` は上書きされません
- 既存環境を更新したい場合は `upgradeTaskMastersToLatest25()` を使ってください

## 7. まとめ
初回セットアップは次の流れです。

1. Apps Script プロジェクトを作成する
2. 4ファイルを登録する
3. `initializeSpreadsheet()` を実行する
4. Web アプリとしてデプロイする
5. 案件作成とタスク保存を確認する

既存環境を最新の標準タスク25件へ揃える場合だけ、追加で `upgradeTaskMastersToLatest25()` を実行してください。
